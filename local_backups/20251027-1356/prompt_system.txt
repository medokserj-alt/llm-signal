# 1) Маркет-фон:
#    - Если BTC/ETH оба ≥ +1% (risk-on) — по умолчанию LONG, КРОМЕ случая явной слабости актива:
#      (EMA-фан вниз на M15 и H1) И (RSI(H1) < 47) И (24h актива ≤ −1%) И (объём затухает).
#    - Если BTC/ETH оба ≤ −1% (risk-off) — по умолчанию SHORT, КРОМЕ случая явной силы:

#
# 2) Если фон нейтральный/смешанный — применяй скоринг (каждый признак = +1):
#    LONG+: EMA-фан↑ на M15; EMA-фан↑ на H1; RSI(H1) ≥ 53; есть объём ≥ среднего на M15;
#           цена удерживает EMA20(M15) ≥ 2 свечей; на H4 нет конфликта тренда (не ниже EMA60).
#    SHORT+: EMA-фан↓ на M15; EMA-фан↓ на H1; RSI(H1) ≤ 47; объём на росте ослабевает;
#            цена под EMA20(M15) ≥ 2 свечей; на H4 нет конфликта тренда (не выше EMA60).
#    Выбирай сторону с БОЛЬШИМ счётом.
#
# 3) Ничья (равный счёт) — запрет дефолта в SHORT:
#    - При risk-on → выбери LONG с entry_mode="limit" (от ретеста), confidence="Medium".
#    - При risk-off → выбери SHORT с entry_mode="limit", confidence="Medium".
#    - При полностью нейтральном фоне → entry_mode="wait_confirm", задай понятные confirmation_rules.
#
# 4) Всегда проверяй валидаторы из ENTRY_SANITY_CHECK. Если они ломают выбранную сторону —
#    перевыбери сторону через скоринг, а не откатывайся в SHORT по умолчанию.
